## 3. コンテナ構成（サービス分割）

| サービス | 役割 | Falcon Sensor | 公開 |
|---|---|---|---|
| **frontend** | チャットUI + AIプロキシ | ✅ falconutil でインストール | Internet公開 |
| **mock-api** | OpenAI互換APIサーバ（走れメロス返答） | ❌ 不要 | Render内部のみ |

- Render上では2つの Web Service として独立デプロイ
- frontend → mock-api は Render の内部URLで通信（`MOCK_API_URL` 環境変数）

## 5. GitHub Actions ワークフロー構成

### ci.yml（CIパイプライン）

```
トリガー: push / pull_request → main, develop

ジョブ:
  1. build
     - Docker build (frontend, mock-api)
  2. fcs-scan
     - CrowdStrike/fcs-action
       - IaC Scan (render.yaml, Dockerfile)
       - Image Scanning (ビルドしたコンテナイメージ)
       - SBOM Generation
```

### deploy.yml（CDパイプライン）

```
トリガー: push → main（ciジョブ成功後）

ジョブ:
  1. deploy
     - Render Deploy Hook を呼び出し
       (RENDER_DEPLOY_HOOK_URL を Secret で管理)
```

## 6. 安全なバックドア（EDRデモ用）

### アイデア：「デバッグコマンド実行エンドポイント」

**仕組み:**
- フロントエンドのUIに隠しページ `/debug` を設ける
- パスワード入力欄があり、正しいパスワード（GitHub Secretで設定）を入力するとコマンド入力フォームが現れる
- 実行できるコマンドは **ホワイトリスト方式** で制限
  - `id`, `whoami`, `uname -a`, `ps aux`, `ls /`, `cat /etc/os-release`, `chgrp 0 /etc/ld.so.preload` のみ許可
  - `chgrp 0 /etc/ld.so.preload` は Falcon EDR アラート発報用のデモコマンド
  - それ以外は拒否（コマンドインジェクション対策）
- コマンド実行はサーバ側で `subprocess.run()` を使い、Falcon Sensorがコマンドラインをキャプチャ

**デモシナリオ:**
1. `/debug` ページにアクセス
2. パスワードを入力してホワイトリストコマンドを実行
3. Falcon Console の Process Timeline / AIDRで実行されたコマンドが記録されていることを確認

**安全性の担保:**
- パスワード保護（総当たり対策にレートリミットも設定）
- ホワイトリスト以外は実行不可
- `shell=False` で実行（シェルインジェクション防止）
- デモ用途であることをUI上に明記

## 7. Render デプロイ構成（render.yaml）

```yaml
services:
  - type: web
    name: melos-frontend
    image:
      url: ghcr.io/cs-japan-se/fcs-cicd-lab-melos-private/melos-frontend:latest
    plan: free
    envVars:
      - key: MOCK_API_URL
        fromService:
          name: melos-mock-api
          type: web
          property: host
      - key: FLASK_SECRET_KEY
        sync: false
      - key: LOGIN_PASSWORD
        sync: false
      - key: DEBUG_PASSWORD
        sync: false
      - key: CS_APP_NAME
        value: melos-frontend
      - key: CS_AIDR_BASE_URL_TEMPLATE
        sync: false
      - key: CS_AIDR_TOKEN
        sync: false

  - type: web
    name: melos-mock-api
    image:
      url: ghcr.io/cs-japan-se/fcs-cicd-lab-melos-private/melos-mock-api:latest
    plan: free
```

## 8. GitHub Secrets 管理すべき環境変数

| Secret名 | 用途 | 設定箇所 |
|---|---|---|
| `FALCON_CLIENT_ID` | Falcon FCS Action 認証 | GitHub Secrets |
| `FALCON_CLIENT_SECRET` | Falcon FCS Action 認証 | GitHub Secrets |
| `FALCON_CID` | Falcon Container Sensor インストール | GitHub Secrets |
| `RENDER_DEPLOY_HOOK_FRONTEND` | Render frontend デプロイトリガー | GitHub Secrets |
| `RENDER_DEPLOY_HOOK_MOCK_API` | Render mock-api デプロイトリガー | GitHub Secrets |
| `FLASK_SECRET_KEY` | Flaskセッション署名キー | Render Environment Variables |
| `LOGIN_PASSWORD` | チャット画面ログインパスワード | Render Environment Variables |
| `DEBUG_PASSWORD` | デバッグページのパスワード | Render Environment Variables |
| `CS_AIDR_BASE_URL_TEMPLATE` | AIDR エンドポイント URL テンプレート | Render Environment Variables |
| `CS_AIDR_TOKEN` | AIDR 認証トークン | Render Environment Variables |

> ⚠️ 上記の値はコード・ファイルに直接記載しないこと。`.gitignore` に `.env` を追加すること。

---

## 9. セキュリティ考慮事項

- APIキー類は全て環境変数経由で注入、コードにハードコードしない
- `.gitignore` に `.env`, `*.key`, `secrets/` を追加
- Render の Internal URL を使い、mock-api をインターネット非公開にする
- チャット画面はログインパスワード（`LOGIN_PASSWORD`）で保護
- デバッグエンドポイントはホワイトリスト + パスワード + レートリミットで保護

## 10. AIDR 連携

チャット画面にトグルスイッチで AIDR ON/OFF を切り替え可能。

| イベント | タイミング | 内容 |
|---|---|---|
| `input` | LLM 呼び出し前 | ユーザーメッセージを AIDR に送信 |
| `output` | LLM 応答後 | ユーザーメッセージ＋AIレスポンスを AIDR に送信 |

- `CS_AIDR_BASE_URL_TEMPLATE` と `CS_AIDR_TOKEN` が未設定の場合は自動的に無効化
- SDK: `crowdstrike-aidr`（Python 3.12+ 必須）
